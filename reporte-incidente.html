Chatbot WhatsApp — Grupo Italocks
Fecha: 9 al 11 de junio de 2026 · Hora peruana (UTC-5)

⚠ Resumen ejecutivo

El bot dejó de recibir mensajes de WhatsApp el 9 de junio a las 7:49 PM (hora Perú) tras la creación accidental de un inbox de WhatsApp Cloud duplicado en Chatwoot. Meta rechazó el intento de sobreescribir el webhook callback y la comunicación entre Meta, Chatwoot y el bot quedó interrumpida. El bot en sí nunca falló — el problema está en la cadena Meta → Chatwoot.

🔍 Causa raíz

El error fatal en los logs de Chatwoot:

[WHATSAPP] Webhook setup failed: Webhook callback override failed:
{"error":{"message":"(#100) Before override the current callback uri,
your app must be subscribed to receive messages for WhatsApp Business Account",
"type":"OAuthException","code":100}}

Traducción: Meta bloqueó el intento de Chatwoot de reconfigurar el callback URL porque la aplicación de Meta no tiene activa la suscripción al campo messages en los Webhook Fields.

Esto ocurrió porque al crear un inbox nuevo con el mismo número, Chatwoot intenta sobreescribir el callback URL registrado, y Meta exige que la app ya tenga la suscripción messages activa antes de permitir el override.

✅ Solución (requiere acceso a Meta Developer)

El cliente o la persona con acceso al Meta Developer App debe hacer esto en 2 minutos:

  1. Entrar a https://developers.facebook.com/apps/
  2. Seleccionar la app de Italocks / Grupo Italocks
  3. Ir a WhatsApp > Configuration
  4. En Webhook Fields, asegurarse de que messages esté tildado (suscrito)
  5. Verificar que el Webhook Callback URL apunte a la URL de Chatwoot y diga Verified
  6. Volver a Chatwoot > Settings > Inboxes > WhatsApp Cloud > editar el inbox existente y guardar. Chatwoot reintentará la conexión.
  7. Verificar que el webhook del bot en Inbox Settings > Configuration > Webhook URL siga apuntando a la URL del bot en EasyPanel.

🛡 Cómo evitar que vuelva a pasar

Acción a evitar Qué hacer en su lugar

NO Tocar configuraciones del inbox si los mensajes dejan de llegar sin entender la causa Revisar los logs de Chatwoot primero para identificar el error real. Si no hay logs de process_payload el problema está entre Meta y Chatwoot
NO Actualizar API key, borrar y recrear, tocar Meta Business sin diagnóstico previo Solo se cambia la API key si Meta la revocó o expiró. Si el inbox existe y está conectado, no tocarlo
NO Hacer cambios sin consultar si no estás seguro El impulso de "pruebo tocando esto a ver si se arregla" suele empeorar el problema

📚 Reglas de oro

  • Siempre editar el inbox existente, nunca crear uno nuevo para el mismo número.
  • Siempre revisar los logs antes de tocar cualquier configuración.
  • Si no hay logs de WhatsApp entrantes (process_payload) en Chatwoot, el problema no es el bot, está en Meta.
  • El bot sólo falla si no ves WEBHOOK RECIBIDO en sus logs de EasyPanel. Si lo ves pero no responde, el problema es interno del bot. Si no hay nada, el problema está aguas arriba.
